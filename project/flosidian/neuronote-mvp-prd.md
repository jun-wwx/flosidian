# NeuroNote - 类 Obsidian 笔记应用 MVP PRD

---

## 1. 产品概述

### 1.1 产品定位

**NeuroNote** 是一款本地优先的**双向链接笔记应用**，帮助用户构建个人知识图谱，实现笔记之间的有机关联，打造"数字第二大脑"。

### 1.2 核心价值主张

- **本地优先**：Markdown 文件存储在本地，数据完全归用户所有
- **双向链接**：让笔记不再是孤岛，而是形成知识网络
- **关系可视化**：通过图谱直观呈现知识结构
- **轻量快速**：无云端依赖，启动即用

### 1.3 目标用户

| 用户类型 | 需求 |
|----------|------|
| 个人知识管理者 | 整理学习笔记、研究资料 |
| 写作者 | 管理素材、构建文章结构 |
| 开发者 | 技术文档、代码笔记 |
| 学生 | 课程笔记、考试复习 |

### 1.4 版本规划

| 版本 | 阶段 | 目标 |
|------|------|------|
| v0.1 | MVP | 核心编辑 + 双向链接 + 反向链接 |
| v0.2 | 迭代 | Graph 图谱 + 标签系统 |
| v0.3 | 完善 | 搜索 + 主题 + 插件基础 |

## 2. 功能需求

### 2.1 核心功能列表

#### F1：Markdown 编辑器
- **F1.1** 实时预览模式（编辑区 + 预览区 split view）
- **F1.2** 语法高亮（代码块、标题、列表、链接等）
- **F1.3** 支持标准 Markdown 语法（GFM）
- **F1.4** 行号显示
- **F1.5** 自动保存（每 3 秒或失去焦点时）

#### F2：双向链接
- **F2.1** 输入 `[[` 触发笔记标题联想
- **F2.2** 点击 `[[笔记名]]` 跳转到对应笔记
- **F2.3** 不存在的链接以特殊样式显示（创建入口）
- **F2.4** 支持别名：`[[笔记名|显示文字]]`
- **F2.5** 链接hover预览

#### F3：反向链接
- **F3.1** 页面底部显示"反向链接"区域
- **F3.2** 列出所有引用当前笔记的笔记列表
- **F3.3** 点击反向链接跳转到引用位置
- **F3.4** 显示引用上下文（引用语句片段）

#### F4：文件管理
- **F4.1** 左侧文件树显示笔记目录结构
- **F4.2** 新建笔记（.md 文件）
- **F4.3** 删除笔记（移到回收站/直接删除）
- **F4.4** 重命名笔记（自动更新相关链接）
- **F4.5** 文件夹创建/删除/重命名

#### F5：Graph 图谱（v0.2）
- **F5.1** 力导向图显示所有笔记节点
- **F5.2** 节点大小基于链接数量（越多人气越高）
- **F5.3** 点击节点跳转到对应笔记
- **F5.4** 拖拽平移 + 滚轮缩放
- **F5.5** 高亮当前笔记及其关联节点

#### F6：标签系统（v0.2）
- **F6.1** 支持 `#标签` 语法
- **F6.2** 侧边栏显示所有标签及笔记数量
- **F6.3** 点击标签过滤显示相关笔记

#### F7：快速切换（v0.2）
- **F7.1** `Ctrl/Cmd + K` 打开命令面板
- **F7.2** 支持搜索笔记标题
- **F7.3** 最近打开的笔记列表

## 3. 用户交互设计

### 3.1 界面布局

```
┌─────────────────────────────────────────────────────────────────┐
│  标题栏 (Title Bar)                                    ─ □ ✕  │
├─────────┬───────────────────────────────────────┬─────────────┤
│         │                                       │             │
│  文件树  │          编辑区 / 预览区              │  反向链接   │
│  侧边栏  │                                       │   面板      │
│  (220px)│         (flex: 1)                     │  (240px)    │
│         │                                       │             │
│         │                                       │             │
│         │                                       │             │
├─────────┴───────────────────────────────────────┴─────────────┤
│  状态栏：字数 | 当前笔记路径 | 保存状态                          │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 关键交互流程

#### 创建新笔记
1. 点击文件树顶部 "+" 按钮 或 `Ctrl/Cmd + N`
2. 输入笔记名称
3. 自动创建 .md 文件并打开编辑
4. 左侧文件树刷新

#### 双向链接操作
1. 在编辑区输入 `[[`
2. 弹出联想下拉框，显示已有笔记标题
3. 选择或继续输入后按 `Enter`
4. 链接以 `[[笔记名]]` 形式插入
5. `Ctrl/Cmd + 点击` 跳转到目标笔记

#### 链接不存在的笔记
1. 输入 `[[新笔记名]]`
2. 链接显示为红色（未创建）
3. `Ctrl/Cmd + 点击` 触发创建确认
4. 自动创建笔记并跳转

#### 查看反向链接
1. 打开任意笔记
2. 右侧面板自动显示反向链接列表
3. 点击任意引用项跳转到对应笔记的引用位置

## 4. 技术方案

### 4.1 技术栈

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 框架 | **Electron** + React | 跨平台桌面应用，生态成熟 |
| 语言 | **TypeScript** | 类型安全 |
| 编辑器 | **Monaco Editor** | VS Code 同款，体验好 |
| Markdown解析 | **remark** / **markdown-it** | 成熟稳定 |
| 图谱 | **react-force-graph** | 轻量级力导向图 |
| 状态管理 | **Zustand** | 轻量、简单 |
| 样式 | **Tailwind CSS** | 快速开发 |
| 构建 | **Vite** | 快速热更新 |

### 4.2 数据结构

#### 笔记文件结构
```
vault/
├── folder1/
│   ├── note1.md
│   └── note2.md
├── folder2/
│   └── note3.md
└── note4.md
```

#### 笔记元数据（内存中维护）
```typescript
interface NoteMeta {
  path: string;           // 文件路径
  title: string;           // 笔记标题（从文件名或 YAML 提取）
  links: string[];         // 正向链接 ["/folder/note1", "/note4"]
  backLinks: string[];     // 反向链接（计算得出）
  tags: string[];         // 标签 ["#tag1", "#tag2"]
  createdAt: number;      // 创建时间
  modifiedAt: number;     // 修改时间
}
```

#### 链接解析规则
- 匹配正则：`\[\[([^\]|]+)(?:\|[^\]]+)?\]\]`
- 示例：`[[笔记]]` → 目标：`笔记.md`
- 示例：`[[笔记|别名]]` → 目标：`笔记.md`，显示：`别名`

### 4.3 核心模块设计

```
src/
├── main/                    # Electron 主进程
│   ├── index.ts            # 入口
│   ├── ipc-handlers.ts     # IPC 通信处理
```

## 6. 非功能需求

### 6.1 性能
- 应用冷启动 < 3 秒
- 笔记切换响应 < 100ms
- 支持 1000+ 笔记文件

### 6.2 安全
- 所有数据存储在用户本地目录
- 不上传任何数据到云端
- 不收集任何用户行为数据

### 6.3 可扩展性
- 预留插件系统接口
- 支持自定义 CSS 主题
- 未来可接入云存储同步
